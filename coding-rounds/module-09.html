Step-by-Step Guide ▼

U - Understand (2-3 min)

  • Restate the problem in your own words
  • Clarify inputs, outputs, constraints
  • Ask about edge cases
  • Confirm examples

"Let me make sure I understand..."

M - Match (1-2 min)

  • Identify problem patterns
  • Consider similar problems solved before
  • Think about applicable data structures

"This reminds me of..."

P - Plan (3-5 min)

  • Discuss brute force first
  • Propose optimized approach
  • Explain time/space complexity
  • Get interviewer buy-in

"Before I start coding, here's my approach..."

I - Implement (15-20 min)

  • Write clean, readable code
  • Narrate as you code
  • Use meaningful names
  • Handle edge cases

R - Review (3-5 min)

  • Walk through with an example
  • Check for bugs
  • Test edge cases

"Let me trace through this..."

E - Evaluate (2-3 min)

  • Discuss final complexity
  • Mention possible optimizations
  • Ask if interviewer wants to explore anything

Communication Patterns

Phrases to Use

Situation What to Say
Starting "Let me make sure I understand the problem correctly..."
Thinking "I'm thinking about this... one approach could be..."
Before coding "Does this approach sound reasonable before I start coding?"
Stuck "I'm not sure about X. Could you give me a hint?"
Getting hint "That makes sense! So I should..."
Finishing "Let me trace through with an example to verify..."
