The "AI Engineer" Mental Model

An AI Engineer is like a solutions architect for AI. You're not inventing new models - you're figuring out how to use existing models to solve real business problems.

Analogy: If ML Engineers are the people who build car engines, AI Engineers are the people who design and build entire cars using those engines. You need to understand engines, but your job is making cars that people actually want to drive.

Why AI Engineering Emerged

Before 2022, building AI products required training your own models - a specialized, expensive task. With GPT-3.5, Claude, and other foundation models, a new role emerged:

  • Old World: Train models, need PhDs, expensive GPUs, months of work
  • New World: Use APIs, need software engineering skills, focus on applications

Skills Breakdown by Priority

Tier 1: Must-Have Foundation (Week 1-2 Focus)
  • Python proficiency: async/await, type hints, decorators, context managers
  • API integration: OpenAI, Anthropic, and other LLM APIs
  • Basic RAG: Embeddings, vector stores, retrieval basics
  • Git workflow: Branches, PRs, code review
Tier 2: Core Competencies (Week 3-6 Focus)
  • Advanced RAG: Chunking strategies, reranking, hybrid search
  • Prompt engineering: System prompts, few-shot, chain-of-thought
  • Agent frameworks: LangChain, LangGraph, tool calling
  • Evaluation: Building evals, measuring quality
Tier 3: Differentiators (Week 7-10 Focus)
  • Production systems: Caching, rate limiting, error handling
  • Cost optimization: Model routing, token management
  • Multi-agent orchestration: Complex workflows, state management
  • Observability: LangSmith, custom tracing, debugging
Tier 4: Advanced (Ongoing)
  • Fine-tuning: When and how to fine-tune models
  • Model internals: Understanding transformers, attention
  • Research awareness: Reading papers, staying current

3. Building a Compelling AI Portfolio

The Portfolio That Gets You Hired ▼

Portfolio Strategy

graph LR subgraph "Portfolio Components" A[1 Deep Project] --> B[2-3 Focused Projects] B --> C[Blog Posts] C --> D[Open Source] D --> E[Social Proof] end

The Ideal AI Engineer Portfolio

1. One "Hero" Project (Essential)

This is your capstone - a complete, deployed project that demonstrates end-to-end skills.

  • Solves a real problem someone cares about
  • Deployed and accessible (demo URL or video)
  • Includes evaluation metrics and benchmarks
  • Well-documented with architecture diagrams
  • Handles edge cases gracefully
  • Cost analysis included
2. Two-Three Focused Projects

Each should showcase a different skill:

  • RAG Project: Document Q&A, knowledge base, or search system
  • Agent Project: Tool-using AI, workflow automation
  • Fine-tuning Project: Custom model for specific task (optional but impressive)
3. Technical Writing

Shows you can communicate complex ideas:

  • 2-3 blog posts explaining AI concepts or project learnings
  • Tutorial or guide that helps others
  • Post-mortem of a failure (shows maturity)
4. Open Source Contributions
  • PRs to LangChain, LlamaIndex, or other AI libraries
  • Bug fixes count! Don't need to be major features
  • Documentation improvements are valuable
Portfolio Anti-Patterns
  • Tutorial recreations: "I followed a YouTube tutorial" projects don't impress
  • No metrics: "It works" isn't enough - show numbers
  • Over-engineered: Complex architecture for simple problems looks naive
  • No demo: If interviewer can't see it, it doesn't count
  • Abandoned projects: Multiple half-finished repos signal poor execution

  • Speed focus: Build something useful in a weekend, then iterate
  • User feedback: Show real users and their feedback

Common Interview Questions and How to Answer ▼

Technical Questions

Q: "Explain how RAG works and when you'd use it vs. fine-tuning."

What they're testing: Do you understand the fundamental tradeoff?

Good answer structure:
1. Define RAG: Retrieval-Augmented Generation - give the model relevant context at runtime
2. When to use RAG: Dynamic data, need citations, privacy constraints
3. When to fine-tune: Change model behavior, domain-specific language, no retrieval latency
4. Real example from your work showing this decision

Q: "How would you evaluate the quality of an LLM application?"

What they're testing: Production mindset, beyond "it works"

Good answer structure:
1. Define metrics: Accuracy (vs ground truth), groundedness (for RAG), user satisfaction
2. Automated evals: LLM-as-judge, deterministic checks, regression tests
3. Human evals: When automated isn't enough, sampling strategy
4. Production monitoring: Latency, cost, error rates, user feedback loops

Q: "Design a customer support chatbot for a bank."

What they're testing: System design thinking, safety awareness

Key points to hit:
1. Clarify requirements: Volume, latency needs, integration with existing systems
2. Architecture: RAG for knowledge base, guardrails for financial advice
3. Safety: Input/output filters, PII handling, escalation to humans
4. Evaluation: Measure resolution rate, CSAT, hallucination rate
5. Cost: Model selection, caching, when to use cheaper models

Q: "Tell me about a time an AI system you built failed."

What they're testing: Maturity, learning from failures

STAR format:
Situation: "I built a RAG system for legal documents..."
Task: "Needed to answer questions with legal accuracy..."
Action: "Discovered hallucinations in 15% of responses. Added citation verification, groundedness checks, and user feedback..."
Result: "Reduced hallucinations to 2%, learned to always have evals before launch"

Questions to Ask the Interviewer

  • "What does the AI infrastructure look like? Are you using hosted APIs, self-hosted models, or both?"
  • "How do you evaluate AI quality? What metrics matter most?"
  • "What's the biggest challenge the AI team is facing right now?"
  • "How do you handle AI safety and responsible deployment?"
  • "What does a typical project look like from idea to production?"

Note: AI Engineer salaries have been inflated by 15-30% compared to equivalent software engineering roles due to high demand and limited supply. This may normalize as more engineers upskill.

Factors That Affect Compensation

  • Location: SF/NYC pay 20-40% more than other US cities
  • Company stage: Big Tech pays more cash, startups offer equity upside
  • Specialization: Inference optimization, safety, multi-modal can command premiums
  • Previous AI experience: Real production AI experience is rare and valued

Negotiation Tips

  • Know your market: Use levels.fyi, Blind, LinkedIn salary insights
  • Get multiple offers: Best leverage is a competing offer
  • Negotiate everything: Base, bonus, equity, sign-on, start date
  • Focus on total comp: Base is just one piece
  • Don't reveal current salary: Many states ban this question anyway

8. Building Your Personal Brand

Why Personal Brand Matters in AI ▼

The Power of Being Known

In a competitive job market, being "the person who does X" gets you opportunities that job applications never will.

Reality: The best AI jobs are never posted. They go to people the hiring manager already knows or has heard of through the community.

Brand Building Metrics to Track

500+ Twitter followers 100+ LinkedIn connections in AI 3+ Blog posts/year 1+ Talks/presentations

Content Strategy

What to Share
  • Build in public: Share progress on projects, including failures
  • Explain things: "How I built X" posts perform well
  • Curate: Share interesting papers/tools with your take
  • Help others: Answer questions, create tutorials
