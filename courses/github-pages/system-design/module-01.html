Load Balancing

Algorithms & Strategies ▼
flowchart TD C[Clients] --> LB[Load Balancer] LB --> S1[Server 1] LB --> S2[Server 2] LB --> S3[Server 3]

Common Algorithms

  • Round Robin: Rotate through servers sequentially
  • Weighted Round Robin: Based on server capacity
  • Least Connections: Route to least busy server
  • IP Hash: Consistent routing for same client
  • Consistent Hashing: Minimize remapping on changes

L4 vs L7 Load Balancers

  • L4 (Transport): TCP/UDP level, faster, less intelligent
  • L7 (Application): HTTP level, can inspect content, more flexible

Caching

Caching Strategies ▼
flowchart LR subgraph "Cache-Aside" A1[App] -->|1. Check| C1[Cache] C1 -->|2. Miss| A1 A1 -->|3. Fetch| D1[DB] A1 -->|4. Update| C1 end

Cache Patterns

  • Cache-Aside: App manages cache (most common)
  • Read-Through: Cache fetches from DB on miss
  • Write-Through: Write to cache, cache writes to DB
  • Write-Behind: Async write to DB

Eviction Policies

  • LRU: Least Recently Used
  • LFU: Least Frequently Used
  • TTL: Time To Live expiration
