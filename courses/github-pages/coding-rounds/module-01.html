Big-O Notation

Understanding Complexity ▼

Big-O describes the upper bound of algorithm growth rate as input size increases.

graph TD subgraph "Common Complexities" A["O(1) - Constant"] B["O(log n) - Logarithmic"] C["O(n) - Linear"] D["O(n log n) - Linearithmic"] E["O(n²) - Quadratic"] F["O(2ⁿ) - Exponential"] end A --> B --> C --> D --> E --> F

Key Rules

  • Drop constants: O(2n) → O(n)
  • Drop lower-order terms: O(n² + n) → O(n²)
  • Different inputs use different variables: O(a + b), not O(n)

Core Data Structures

Arrays vs Linked Lists ▼
Operation Array Linked List
Access O(1) O(n)
Search O(n) O(n)
Insert (beginning) O(n) O(1)
Insert (end) O(1)* O(1)**
Delete O(n) O(1)***

* Amortized. ** With tail pointer. *** With reference to node.

Hash Tables ▼

Hash tables provide O(1) average-case operations through key-to-index mapping.

# Hash table operations (average case)
d = {}
d[key] = value    # O(1) insert
d[key]            # O(1) lookup
del d[key]        # O(1) delete
key in d          # O(1) membership

# Common pattern: Two Sum
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

Stacks & Queues ▼
flowchart LR subgraph "Stack (LIFO)" S1[Push] --> S2[Top] S2 --> S3[Pop] end subgraph "Queue (FIFO)" Q1[Enqueue] --> Q2["..."] --> Q3[Dequeue] end 
# Stack: Valid Parentheses
def is_valid(s):
    stack = []
    mapping = {')': '(', '}': '{', ']': '['}

    for char in s:
        if char in mapping:
            if not stack or stack.pop() != mapping[char]:
                return False
        else:
            stack.append(char)

    return len(stack) == 0

from collections import deque

queue.append(1)     # enqueue
queue.popleft()     # dequeue
