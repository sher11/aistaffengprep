Databricks, Snowflake Preparation

Learning Objectives

  • Demonstrate deep systems knowledge
  • Handle panel presentation format
  • Discuss trade-offs effectively
  • Show domain expertise

Databricks

Focus Areas ▼
  • Deep understanding of coding and systems
  • Practical, non-LeetCode problems
  • Distributed systems knowledge
  • Data processing concepts
Databricks Question Examples ▼
  • Shortest path with multiple transport modes
  • Snapshot set (HashSet with versioning)
  • Concurrent data writer (thread-safe)
  • Key-value store with locking

Snowflake

Interview Structure ▼
flowchart LR A[Tech Screen
2 questions] --> B[Onsite] B --> C[Panel Presentation
30 min, 5-6 interviewers] C --> D[Decision]
Panel Presentation Guide ▼

Slide Structure (10-12 slides):

  1. Title and intro
  2. Context/problem space
  3. Requirements/constraints
  4. Solution overview
  5. Architecture diagram
  6. Key technical decisions
  7. Trade-offs considered
  8. Implementation challenges
  9. Results/metrics
  10. Learnings
Tips: Know exact numbers, be honest about unknowns, practice timing (30 min strict).
